# Truth Space: A Primer

## What is Truth Space?

Truth space is a geometric framework where **valid configurations occupy specific regions defined by mathematical constraints**. It's not a physical space but a conceptual one where the coordinates represent properties of valid solutions.

Think of it like this:
- A 2D plane can represent all possible (x, y) pairs
- But only some pairs satisfy y = x² (a parabola)
- The parabola IS the "truth space" for that constraint

For neural networks and other structured data, truth space is much higher-dimensional, and the constraints are more subtle—but the principle is the same.

### Algebraic Irrationals as Axes

Truth space is organized around algebraic irrationals—numbers that are roots of polynomial equations with integer coefficients:

| Constant | Definition | Approximate Value |
|----------|------------|-------------------|

These aren't arbitrary—they represent fundamental ratios that appear throughout mathematics and nature.

### The Golden Ratio is Special

The golden ratio φ has unique properties:

1. **Self-referential**: φ² = φ + 1
2. **Continued fraction**: φ = 1 + 1/(1 + 1/(1 + ...))
3. **Fibonacci limit**: lim(F_{n+1}/F_n) = φ
4. **Optimal packing**: φ-based spirals minimize overlap

These properties make φ the natural "unit" of truth space.

### Fractal Structure

Truth space exhibits self-similarity at multiple scales:

- Zoom in on any region → similar structure appears
- The Sierpiński gasket (dimension ≈ 1.585) appears repeatedly

This fractal nature means finite rules can describe infinite structure.

## How We Use Truth Space for Compression

### The Key Insight

Neural network weights don't occupy arbitrary positions—they cluster near truth space grid points. The deviation from the nearest grid point is small and structured.

### The φ-Grid

We build a grid based on powers of φ:

```
grid = {0, ±φ^(-1), ±φ^(-2), ±φ^(-3), ...}
```

This grid is:
- **Deterministic**: Same parameters → same grid
- **Self-similar**: Zooming in reveals same structure
- **Optimal**: φ-spacing minimizes maximum deviation

### Compression as Truth Space Projection

Compressing a weight:
1. Find nearest grid point (truth space position)
2. Compute deviation (distance from grid)
3. Store (position, deviation)

Decompressing:
1. Rebuild grid (FREE—computed from φ)
2. Look up position
3. Add deviation
4. Recover original weight

### Why This Works

The grid captures the "expected" structure of valid weights. The deviation captures what makes THIS model unique. Since most of the structure is predictable, we only need to store the unique part.

## Group Theory in Truth Space

### Truth Space as a Group

We can view truth space through group theory:

- **Elements**: Points in truth space
- **Operation**: Transformation between points
- **Identity**: Origin (zero deviation)
- **Inverse**: Negation

### φ-Based Generators

The multiplicative group generated by φ:

```
G = {..., φ^(-2), φ^(-1), 1, φ, φ², ...}
```

This group acts on truth space by scaling. Moving from one grid point to another is a group action.

### Navigation via Group Actions

To go from grid point g to target value v:
1. Compute deviation d = v - g
2. Express d as group action: d ≈ φ^k × ε
3. Store k (which power of φ) and ε (residual)

This is more efficient than storing d directly because k is an integer (few bits) and ε is small.

### Clock Phases

We encode position using "clock phases":

```
phase(x) = (x × ratio) mod 1
```

Different ratios (φ, √2, etc.) give different phases. Together they form a multi-dimensional clock that uniquely identifies position in truth space.
